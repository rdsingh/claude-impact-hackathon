The Problem

  • San Diego generates 6 million+ civic records across 3 data sources
  • 311 requests, code enforcement, and police calls live in separate silos
  • No unified way to measure or compare neighborhood quality of life
  • Residents and city staff can't track if conditions are improving or declining
2 / 8

Our Solution

  • Combine 3 city datasets into a single Quality of Life (QoL) score per neighborhood
  • Classify events as critical vs. non-critical with weighted scoring
  • Visualize on an interactive ArcGIS map with 135 PD beat polygons
  • Animate QoL trends over time with play/pause time slider (2018-2025)

Architecture

CSV/Shapefile Data (6M+ records) | v Preprocessing Scripts (Python) - classify events (critical/non-critical) - spatial joins (point-in-polygon) - QoL scoring (weighted, relative to city avg) | v MCP Server + REST API (FastMCP / Starlette) - GeoJSON, filters, ArcGIS OAuth tokens | v Frontend (ArcGIS Maps SDK v4.29) - Interactive map, time slider, filters
